<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Font Import (Google Fonts) | External third-party font source; CDN-delivered |
| Component Inline Styles | User-controlled styling applied at render time |
| CSS Variable Resolution | Browser CSS engine interprets variable cascading |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-typography-001 | I (Information Disclosure) | Google Fonts import | accept | Jost is public font; no secrets in import URL. No PII exposure from font selection. |
| T-01-typography-002 | D (Denial of Service) | Font loading failure | mitigate | Fallback font stack (Helvetica, sans-serif) ensures text renders if Google Fonts CDN is unavailable. Test via network throttling. |
| T-01-typography-003 | T (Tampering) | Component inline styles | accept | Font family values are hardcoded in source; no user input modifies fontFamily property. CSS variable references are resolved at build time. |

## Notes

No authentication, authorization, or cryptography involved in typography changes. Primary risk is font loading failure — mitigated by fallback font stack in CSS variable definition.
</threat_model>
